# Migration history

These 7 files are the exact SQL Supabase has recorded as already applied to the live project (`dzqyqpuhxdrrpipbehpk`), pulled directly from `supabase_migrations.schema_migrations` and committed here for the first time. Before this, this history existed only inside Supabase — this folder is the fix for that gap (`ARCHITECTURE_REDESIGN_PROPOSAL.md` §1.2/1.4, and roadmap item M5 in `docs/19_Future_Roadmap.md`).

**Every schema change from this point on should be written as a new, timestamped file here first, then applied** — so the repository and the live database never drift apart again.
